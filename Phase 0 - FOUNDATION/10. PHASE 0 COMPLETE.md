## üí° PRO TIPS

### Tip 1: Start with Database
The database is your foundation. Get it right first.
```bash
# Day 15: Just focus on this
1. Copy Prisma schema from DATABASE_SPECIFICATION.md
2. Run migration
3. Run seed
4. Open Prisma Studio (npx prisma studio)
5. Verify data looks correct
```

### Tip 2: Test as You Go
Don't wait until the end. Write tests DURING development.
```bash
# After implementing /register endpoint
1. Write unit test (auth.service.test.ts)
2. Write integration test (auth.controller.test.ts)
3. Run tests (npm test)
4. Fix until green
5. Commit
```

### Tip 3: Use Postman/Insomnia
Test API endpoints manually before connecting frontend.
```bash
# Create collection with all 44 endpoints
1. Import API_SPECIFICATION.md examples
2. Test each endpoint after implementation
3. Save successful requests
4. Use for debugging
```

### Tip 4: Frontend Last
Backend first = easier debugging. You can test API independently.
```bash
# Backend Week 3-6
- Test with Postman/Insomnia
- Write automated tests
- Fix bugs in isolation

# Frontend Week 7-8
- Connect to working API
- Focus on UI/UX
- Less debugging
```

### Tip 5: Commit Often
Small commits = easier rollback if something breaks.
```bash
# Good commit practice
git commit -m "feat(auth): add register endpoint"
git commit -m "test(auth): add register tests"
git commit -m "feat(auth): add login endpoint"
git commit -m "test(auth): add login tests"

# Bad commit practice
git commit -m "add auth module" # Too big, hard to rollback
```
