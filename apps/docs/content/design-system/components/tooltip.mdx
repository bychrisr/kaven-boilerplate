# Tooltip

> Componente de feedback flutuante utilizado para clarificar o propósito de elementos da interface sem desordenar o visual.

## Visão Geral

O Tooltip do Kaven UI v2 foi completamente reconstruído utilizando **Radix UI Primitives** para máxima acessibilidade e **Portals** para garantir que o conteúdo nunca seja cortado por containers pais (z-index issue solved). Visualmente, adota o padrão **Glassmorphism** com suporte nativo a Dark Mode e animações suaves.

**Principais Melhorias (v2):**

- ✅ **Portal Nativo:** Renderiza diretamente no `body` para evitar problemas de `overflow: hidden`.
- ✅ **Z-Index Gerenciado:** Configurado como `z-[100000]` para sobrepor Modals e Sheets.
- ✅ **Wrapper Simplificado:** API de componente único para uso rápido.
- ✅ **Visual Premium:** Efeito de vidro fosco com bordas sutis e sombras profundas.

## Uso Simplificado (Recomendado)

Para a maioria dos casos, utilize o wrapper `<Tooltip>` que encapsula toda a complexidade dos primitivos.

```tsx
import { Tooltip } from '@/components/ui/tooltip';

<Tooltip content="Visualize os detalhes completos desta transação">
  <Button variant="outline">Detalhes</Button>
</Tooltip>;
```

## Uso Avançado (Primitivos)

Para controle total sobre o estado de abertura, delays customizados ou conteúdo complexo, use os componentes primitivos exportados.

```tsx
import {
  TooltipProvider,
  TooltipRoot,
  TooltipTrigger,
  TooltipContent,
} from '@/components/ui/tooltip';

<TooltipProvider>
  <TooltipRoot delayDuration={300}>
    <TooltipTrigger asChild>
      <img src="/avatar.png" className="w-10 h-10 rounded-full" />
    </TooltipTrigger>
    <TooltipContent side="right" className="bg-zinc-900 border-zinc-800">
      <div className="flex flex-col gap-1">
        <span className="font-bold">Chris R.</span>
        <span className="text-xs text-muted-foreground">Online agora</span>
      </div>
    </TooltipContent>
  </TooltipRoot>
</TooltipProvider>;
```

Um componente helper pronto para uso que renderiza um ícone de informação típico. Ideal para labels de formulários ou cabeçalhos de cards.

## API Reference

### Wrapper `<Tooltip />`

| Prop       | Tipo                                     | Padrão  | Descrição                                   |
| ---------- | ---------------------------------------- | ------- | ------------------------------------------- |
| `content`  | `string`                                 | -       | O texto a ser exibido dentro do tooltip.    |
| `position` | `'top' \| 'right' \| 'bottom' \| 'left'` | `'top'` | O lado onde o tooltip deve aparecer.        |
| `delay`    | `number`                                 | `200`   | Tempo em ms antes do tooltip abrir.         |
| `disabled` | `boolean`                                | `false` | Se verdadeiro, o tooltip não será exibido.  |
| `children` | `ReactNode`                              | -       | O elemento que dispara o tooltip (Trigger). |

### Helper `<InfoTooltip />`

| Prop        | Tipo     | Padrão | Descrição                            |
| ----------- | -------- | ------ | ------------------------------------ |
| `content`   | `string` | -      | O texto informativo.                 |
| `className` | `string` | -      | Classes CSS adicionais para o ícone. |

### Primitivo `<TooltipContent />`

Aceita todas as props do [Radix UI Tooltip.Content](https://www.radix-ui.com/primitives/docs/components/tooltip#content), mais:

| Prop         | Tipo     | Padrão | Descrição                               |
| ------------ | -------- | ------ | --------------------------------------- |
| `sideOffset` | `number` | `4`    | Distância entre o trigger e o conteúdo. |
| `className`  | `string` | -      | Classes para estilização customizada.   |

## Troubleshooting

### Tooltip cortado ou não aparece?

Se você estiver usando o wrapper `<Tooltip>` ou os primitivos padrão, o conteúdo é renderizado via **React Portal** (`createPortal`) diretamente no `body`. Isso significa que propriedades `overflow: hidden` ou `z-index` em elementos pais **NÃO** devem afetar a visibilidade do tooltip.

Se ainda assim houver problemas, verifique se o arquivo `layout-client.tsx` ou similar não possui `z-index` global extremo que possa competir com `z-[100000]`.

### Acessibilidade

Este componente segue as diretrizes WAI-ARIA para tooltips:

- O Trigger recebe foco via teclado.
- Pressionar `Esc` (quando aberto via teclado) fecha o tooltip.
- O conteúdo é associado semanticamente ao trigger via `aria-describedby`.
