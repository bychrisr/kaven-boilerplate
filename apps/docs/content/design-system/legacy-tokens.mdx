## ðŸŒˆ Colors (Semantic)

Our color system uses semantic tokens to support Light/Dark modes automatically. **Always use the semantic variable**, not the raw hex.

### Core Structure

| Token                   | Dark Mode (Default) | Use Case                                    |
| ----------------------- | ------------------- | ------------------------------------------- |
| `bg-background`         | `#161C24`           | Main application background (Body)          |
| `bg-sidebar`            | `#161C24`           | Sidebar background (matches body for depth) |
| `bg-card`               | `#212B36`           | Cards, Modals, Dropdowns (Floating surface) |
| `text-foreground`       | `#FFFFFF`           | Primary text content                        |
| `text-muted-foreground` | `#919EAB`           | Secondary text, captions, placeholders      |

### Brand Colors

| Token          | Value (Ref) | Use Case                              |
| -------------- | ----------- | ------------------------------------- |
| `bg-primary`   | `#00A76F`   | Main actions, active states, branding |
| `bg-secondary` | `#8E33FF`   | Secondary actions, highlights         |
| `bg-success`   | `#22C55E`   | Success states, positive trends       |
| `bg-warning`   | `#FFAB00`   | Warning alerts, attention needed      |
| `bg-error`     | `#FF5630`   | Destructive actions, negative trends  |
| `bg-info`      | `#00B8D9`   | Informational badges, charts          |

### Interactive States

| Token             | Description                                  |
| ----------------- | -------------------------------------------- |
| `hover:bg-accent` | Subtle interaction background for list items |
| `bg-muted`        | Disabled states or secondary backgrounds     |
| `border-border`   | Subtle borders (`rgba(145, 158, 171, 0.24)`) |
