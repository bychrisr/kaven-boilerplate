# Colors

> Sistema de cores semântico baseado em oklch para light mode e hex para dark mode, garantindo consistência visual e acessibilidade.

## Quando Usar

- **Branding**: Cores primárias e secundárias da marca
- **Feedback**: Cores semânticas para sucesso, erro, aviso e informação
- **UI**: Cores neutras para backgrounds, textos e bordas
- **Charts**: Cores específicas para visualizações de dados

| `background`       | `#FFFFFF`  | `#161C24`                | Background principal    |
| `foreground`       | `#212121`  | `#FFFFFF`                | Texto principal         |
| `card`             | `#FFFFFF`  | `#212B36`                | Background de cards     |
| `card-foreground`  | `#212121`  | `#FFFFFF`                | Texto em cards          |
| `popover`          | `#FFFFFF`  | `#212B36`                | Background de popovers  |

| `ring`             | `#BDBDBD`  | `rgba(0,167,111,0.3)`    | Focus rings             |

## Dark Mode

O Kaven usa o tema Minimals para dark mode com cores cuidadosamente selecionadas.

```tsx
// Ativar dark mode
<html className="dark">{/* Conteúdo */}</html>
```

### Comparação Light vs Dark

```tsx
// Light Mode
--background: oklch(1 0 0);        // Branco puro
--foreground: oklch(0.145 0 0);    // Quase preto
--primary: oklch(0.205 0 0);       // Preto

// Dark Mode
--background: #161c24;              // Azul escuro profundo
--foreground: #ffffff;              // Branco
--primary: #00a76f;                 // Verde Minimals
```

## Uso em Código

### Tailwind Classes

```tsx
// Backgrounds
<div className="bg-primary">Primary Background</div>
<div className="bg-success-main">Success Background</div>
<div className="bg-muted">Muted Background</div>

// Text
<p className="text-foreground">Primary Text</p>
<p className="text-muted-foreground">Secondary Text</p>
<p className="text-primary">Primary Color Text</p>

// Borders
<div className="border border-border">Default Border</div>
<div className="border border-primary">Primary Border</div>
```

### CSS Variables

```css
.custom-component {
  background-color: hsl(var(--primary));
  color: hsl(var(--primary-foreground));
  border: 1px solid hsl(var(--border));
}

/* Dark mode específico */
.dark .custom-component {
  background-color: var(--primary); /* #00a76f */
}
```

### Inline Styles

```tsx
<div
  style={{
    backgroundColor: 'hsl(var(--primary))',
    color: 'hsl(var(--primary-foreground))',
  }}
>
  Custom Styled
</div>
```

## Acessibilidade

### Contraste de Cores

Todas as combinações de cores seguem WCAG 2.1 AA:

| Combinação                        | Contraste | Status |
| --------------------------------- | --------- | ------ |
| `foreground` / `background`       | 16:1      | ✅ AAA |
| `primary` / `primary-foreground`  | 12:1      | ✅ AAA |
| `muted-foreground` / `background` | 4.8:1     | ✅ AA  |
| `error-main` / `white`            | 4.5:1     | ✅ AA  |

### Testes de Contraste

```tsx
// Sempre teste combinações de cores
<Button className="bg-primary text-primary-foreground">
  {/* Contraste garantido: 12:1 */}
  Acessível
</Button>

// ❌ Evite
<Button className="bg-primary-light text-white">
  {/* Contraste insuficiente */}
  Não Acessível
</Button>
```

## Melhores Práticas

### ✅ Fazer

- **Use cores semânticas** para feedback (success, error, warning, info)
- **Mantenha consistência** usando tokens CSS ao invés de valores hardcoded
- **Teste em dark mode** sempre que usar cores customizadas
- **Verifique contraste** com ferramentas como WebAIM Contrast Checker
- **Use opacity** para variações sutis: `bg-primary/10`

### ❌ Não Fazer

- ❌ **Não use hex/rgb direto** - sempre use CSS variables
- ❌ **Não ignore dark mode** - teste todas as cores
- ❌ **Não use cores aleatórias** - siga a paleta estabelecida
- ❌ **Não sacrifique acessibilidade** por estética
- ❌ **Não misture sistemas** (oklch no light, hex no dark está OK)

## Customização

### Adicionar Nova Cor

```css
/* apps/admin/styles/theme.css */
:root {
  --custom-color: oklch(0.7 0.2 150);
}

.dark {
  --custom-color: #10b981;
}
```

```tsx
// Usar no Tailwind
<div className="bg-[hsl(var(--custom-color))]">Custom Color</div>
```

### Modificar Paleta Existente

```css
/* Sobrescrever primary */
:root {
  --primary: oklch(0.5 0.3 200); /* Azul ao invés de verde */
}

.dark {
  --primary: #3b82f6;
}
```
