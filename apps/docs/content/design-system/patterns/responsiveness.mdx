# üì± Responsiveness & Mobile Patterns

Best practices for ensuring the Admin Dashboard works seamlessly across devices.

## üìê Mobile-First Strategy

We code for mobile first, then enhance for larger screens using Tailwind breakpoints.

### 1. Layout Structure

- **Mobile (< 1024px):**
  - Sidebar is **hidden** by default.
  - Hamburger menu toggles a **Drawer Overlay**.
  - Content takes full width (`w-full`, `ml-0`).
- **Desktop (>= 1024px):**
  - Sidebar is **fixed** (always visible).
  - Sidebar pushes content via margin (`ml-[280px]` or `ml-[88px]`).
  - Hamburger menu is hidden.

### 2. Grid System Strategy

To ensure readability on small screens, we strictly control grid columns.

| Component        | Mobile        | Tablet        | Desktop          |
| ---------------- | ------------- | ------------- | ---------------- |
| **Metric Cards** | `grid-cols-1` | `grid-cols-1` | `md:grid-cols-3` |
| **Charts**       | `grid-cols-1` | `grid-cols-1` | `lg:grid-cols-3` |
| **Forms**        | `grid-cols-1` | `grid-cols-2` | `grid-cols-2`    |

> **Why?** Forcing `grid-cols-1` on mobile prevents squashed content and broken charts.

### 3. Padding & Spacing

We reduce density on mobile to maximize screen real estate.

- **Container Padding:**
  - Mobile: `p-4` (Compact)
  - Desktop: `p-6` (Airy)
- **Gap:**
  - Mobile: `gap-4`
  - Desktop: `gap-6`

## üß© Components Behavior

### Sidebar (Mobile Drawer)

- **Trigger:** Hamburger icon in Header.
- **Behavior:** Slides in from left (`translate-x-0`).
- **Overlay:** Backsplash (`bg-black/50`) with backdrop blur.
- **State:** Controlled via global `useUIStore`.

### Cards

- **Mobile:**
  - Reduced padding (`p-4`).
  - Smaller icons (`w-5 h-5`).
  - Font scaling (`text-2xl`).
- **Desktop:**
  - Standard padding (`p-6`).
  - Standard icons (`w-6 h-6`).
  - Large typography (`text-3xl`).

### Tenant Switcher

- **Mobile:**
  - **Compact Mode:** Shows only Avatar + Chevron (saves header space).
- **Desktop:**
  - **Full Mode:** Shows Avatar + Tenant Name + Role + Chevron.

## ‚úÖ Checklist for Responsive Dev

When adding a new feature:

1. [ ] Check layout on **375px** (iPhone SE/Mini) width.
2. [ ] Ensure horizontal scroll does **NOT** appear (use `overflow-x-hidden` if needed).
3. [ ] Verify touch targets are at least **44px**.
4. [ ] Test Sidebar open/close interaction on mobile.
5. [ ] Ensure text doesn't truncate excessively on small screens.
