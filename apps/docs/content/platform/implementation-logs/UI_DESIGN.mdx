# Documentação de UI/UX - Módulo de Autenticação

## Visão Geral

Este documento detalha o redesign das páginas de autenticação do Kaven Admin, alinhando-as com o estilo "Dark Glassmorphism" inspirado no template Minimals UI. O objetivo foi criar uma experiência de login moderna, consistente e visualmente premium.

## Design System Implementado

### 1. Tema & Cores

- **Background Principal:** `#161C24` (Dark/Black) com "blobs" de gradiente sutis (`primary-main/20` e `primary-dark/20`) para profundidade.
- **Surface (Cards):** `#212B36` com borda sutil `gray-700/50` e sombra pronunciada.
- **Tipografia:** Títulos em `white` (Bold) e textos de apoio em `gray-400` ou `gray-600`.

### 2. Layout (AuthLayout)

O layout foi centralizado para focar na tarefa do usuário (login/cadastro).

- **Componente:** `AuthLayout` (`apps/admin/components/layout/auth-layout.tsx`)
- **Mudanças:** Removida a divisão 50/50 anterior. Agora o conteúdo flutua centralizado sobre o background animado.

### 3. Componentes de UI Atualizados

#### TextField

- **Local:** `apps/admin/components/ui/text-field.tsx`
- **Atualização:** Adicionado suporte a `labelClassName` para permitir overrides de cor do label em fundos escuros.
- **Estilo Padrão (Dark):** Background `#161C24`, Text `white`, Border `gray-700`.

#### Botões Sociais

- Convertidos para icon-buttons (Google, Github, Twitter).
- **Estado Atual:** Desabilitados (`disabled`, `opacity-50`, `cursor-not-allowed`) até que a integração de backend seja realizada.

## Telas Refatoradas

1.  **Login (`/login`)**
    - Formulário em card escuro.
    - Input de senha com toggle de visibilidade.
    - Links de "Forgot password" e "Register" integrados.

2.  **Cadastro (`/register`)**
    - Campos de Nome e Sobrenome separados visualmente (concatenados no envio).
    - Medidor de força de senha ajustado para tema escuro.
    - Termos de uso simplificados no footer do form.

3.  **Recuperação de Senha (`/forgot-password` & `/reset-password`)**
    - Fluxos de "Esqueci minha senha" e "Redefinir senha" unificados visualmente.
    - Ícones grandes e feedback de sucesso integrado no próprio card.

4.  **Verificação de Email (`/verify-email`)**
    - Feedback visual claro para estados: `Verifying` (Spinner), `Success` (Check), `Error` (X).
